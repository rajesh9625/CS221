<p class="small-heading"><b>Data Set Information:</b></p>
<p class="normal">The source of the data is the raw measurements from a Nintendo PowerGlove. It was interfaced through a PowerGlove Serial Interface to a Silicon Graphics 4D/35G workstation. <br><br>This glove definitely falls into the category of "cheap and nasty". Position information is calculated on the basis of ultrasound emissions from emitters the glove to a 3-microphone "L-Bar" that sits atop a monitor. There are two emitters on the glove; and three receivers. This allows the calculation of 4 pieces of information: x (left/right), y (up/down), z (backward/forward), and roll (is the palm pointing up or down?). x, y and z are measured with 8 bit accuracy. "x, y, z" should not be taken to be the normal 3-dimensional orthogonal basis. In particular, 1 unit in the z direction is not of similar distance to 1 unit in the x or y directions. These x, y, z positions are relative to a calibration point which is when the palm is resting on the seated signer's thigh. Roll is 4 bits. <br><br>The data is susceptible to occasional "spikes" caused by random ultrasound noise. Median filters have been found to be beneficial in solving this problem. <br><br>Finger bend is generated by conductive bend sensors on the first four fingers. Values vary between 0 (straight) and 3 (fully bent). Accuracy is 2 bits. The gloves automatically apply a hysteresis filter on these bend sensors. At best, these measurements should be treated sceptically. <br><br>See past usage for a more detailed discussion on the data collection methodology. <br><br>The data was collected from five signers: <br> <br>Signer -- Description -- Sessions -- Total samples/sign<br><br> Adam -- Sign linguist - PhD completed in area. -- 2 -- 8<br> Andrew -- Natural signer - signing since youth -- 3 -- 8<br> John -- Professional Auslan interpreter -- 5 -- 18<br> Stephen -- Professional Auslan interpreter -- 4  -- 16<br> Waleed -- The researcher. Novice signer -- 4 -- 20<br><br>Each session was taken at a different time, after a break, etc. <br><br>The "adam" dataset were sampled in a fixed order -- this means that they are subject to fatigue effects, etc. All other datasets were sampled in random order. The "waleed" and "stephen" datasets contain signs that begin with "cal-". These were considered as a means of calibration, but didn't work out too well. <br><br>The data presented is the raw data with no filtering. <br><br>Occasional dropouts in x, y, z values. These can be easily fixed using a median filter. <br><br>Average number of frames per instance is 51, but varies from 30 to 102. <br><br>The data is in a comma separated file containing all of the attributes mentioned above. Each sign sample is stored in a single file. The directory hierarchy is as follows: <br>-Each signer is in a separate directory. <br>-Each session from signer is in a subdirectory. Each session is denoted by a number. <br>-Each sample is in a file named by the sample appended with the number of the sample of that sign. <br><br>The filenames indicate the class. </p>

<br />

<p class="small-heading"><b>Attribute Information:</b></p>
<p class="normal">x:<br>  - Continuous. <br>  - Description: x position between -1 and 1. Units are *approximately* metres. <br>y: <br>  - Continuous. <br>  - Description: y position between -1 and 1. Units are approximately metres. <br>z: <br>  - Continuous. <br>  - Description: z position between -1 and 1. Units are not metres. <br>     This space should not really be treated as linear, although it is safe to<br>     treat it as monotonically increasing. <br>roll:      <br>  - Continuous.<br>  - Description:  roll with 0 meaning "palm down", rotating clcokwise through to a maximum of 1 (not included), which is also "palm down".<br>pitch: <br>   - Has a value of -1, indicating that it is not available for this data. <br>   Should be ignored. <br>yaw: <br>   - Has a value of -1, indicating that it is not available for this data. <br>    Should be ignored. <br>thumb: <br>   - Continuous. <br>   - Description: Thumb bend. has a value of 0 (straight) to 1 (fully bent). <br>fore:<br>   - Continuous. <br>   - Description: Forefinger bend. has a value of 0 (straight) to 1 (fully bent). <br>index:<br>   - Continuous. <br>   - Description: Index finger bend. has a value of 0 (straight) to 1 (fully bent). <br>ring: <br>   - Continuous. <br>   - Description: Ring finger bend. has a value of 0 (straight) to 1 (fully bent). <br>little:      <br>   - In this case, it is a copy of ring bend. Should be ignored. <br>keycode: <br>   - Indicates which key was pressed on the glove. Should be ignored. <br>gs1:      <br>   - glove state 1 Should be ignored. <br>gs2:      <br>   - glove state 2 should be ignored. <br>Receiver values: <br>  - Determines if all receivers received values from all transmitters. A value of 0x3F indicates all receivers received information from all transmitters. Other values indicate this is not the case.<br></p>
